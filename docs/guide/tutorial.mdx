This tutorial walks you through setting up Recyclarr to sync TRaSH Guide quality profiles to Radarr.
By the end, you'll have a working quality profile with custom formats and proper media naming.

:::info No YAML expertise required

You don't need to understand YAML structure or dig into template files. Recyclarr provides
ready-to-use configuration templates that mirror the TRaSH Guides exactly.

:::

## Prerequisites

- **Radarr is running:** You should already have Radarr installed and running (Sonarr setup is similar â€” just use `sonarr` templates)
- **API Key:** Have your Radarr API Key ready (`Settings` â†’ `General` â†’ `Security`)
- **User IDs:** Know your PUID/PGID.
  ```bash
  id -u  # PUID
  id -g  # PGID
  ```

## Step 1: Install Recyclarr

Add the Recyclarr service to your existing `docker-compose.yml` file, or create a new one.

```yml
services:
  recyclarr:
    image: ghcr.io/recyclarr/recyclarr:latest
    container_name: recyclarr
    user: 1000:1000 # Change to your PUID:PGID
    restart: unless-stopped
    # Set the time zone to match your location
    environment:
      - TZ=America/New_York
    volumes:
      - ./config:/config
```

For detailed installation options (networking, tags, native install), see the [Installation
Guide](./installation/index.mdx).

## Step 3: Create Your Configuration

Generate a config file from your chosen template. For example, to create the `HD Bluray + WEB` profile, run:

```bash
docker compose run --rm recyclarr config create --template hd-bluray-web
```

This creates `config/configs/hd-bluray-web.yml` with everything pre-configured.

**To use this configuration:**
- Open the file and update the `base_url` and `api_key`.
  ```yml
  radarr:
    hd-bluray-web:
      base_url: http://radarr:7878  # Your Radarr URL
      api_key: your_api_key_here    # Radarr â†’ Settings â†’ General â†’ API Key
  ```
- (Optional) Add these lines if you want Recyclarr to manage custom formats exclusively (recommended):
  ```yml
      delete_old_custom_formats: true
      replace_existing_custom_formats: true
  ```

That's it! The template already includes the quality definition, quality profile, and custom formats
from the TRaSH Guide.

## Step 5: Run Your First Sync
Now start the container in daemon mode (so it runs automatically every day) and trigger the first sync manually.

```bash
# 1. Start the container
docker compose up -d recyclarr

# 2. Trigger the sync
docker exec recyclarr recyclarr sync
```

:::

Run `docker compose run --rm recyclarr sync` again, and all profiles will be created.

## When to Customize

The templates provide the full TRaSH Guide recommendation. For most users, that's everything you need.

You only need a `custom_formats:` section if you want to:

- **Override a score** â€” Change the default score for a specific custom format
- **Add optional custom formats** â€” Include CFs marked as "optional" in the TRaSH Guides.
  - For example if you want releases with hybrid `DV + HDR10` to be preferred, simply uncomment that section in your config file.
  - See it in action in [Level Up setup](#level-up-the-power-user-config) below.

For customization examples, see the [Configuration Examples](../reference/config-examples.mdx) page.

## Level Up: The "Power User" Config
Ready for the full setup? The configuration below is a "One and Done" setup. It includes:

- **Radarr**: 1080p, 4K, and Remux profiles.
- **Sonarr**: 1080p and 4K profiles.
- **Customizations**: Enabled DV/HDR10+ support.

### Steps
1. Make sure Radarr and Sonarr are running.
2. Check out the templates that you want to combine.
   ```bash
   docker exec recyclarr recyclarr config list templates
   ```
   These are the common ones for Radarr:
    - `hd-bluray-web`
    - `uhd-bluray-web`
    - `remux-web-1080p`
    - `remux-web-2160p`
    - `anime-radarr`

   And these for Sonarr:
    - `web-1080p-v4`
    - `web-2160p-v4`
    - `anime-sonarr-v4`
3. Create config file for all the templates you want to combine.
   ```bash
   docker compose run --rm recyclarr config create --template hd-bluray-web
   docker compose run --rm recyclarr config create --template uhd-bluray-web
   docker compose run --rm recyclarr config create --template remux-web-1080p
   docker compose run --rm recyclarr config create --template remux-web-2160p
   docker compose run --rm recyclarr config create --template web-1080p-v4
   docker compose run --rm recyclarr config create --template web-2160p-v4
   ```
   This will create 6 files in `config/configs` folder. You can combine all their contents into a single `config/recyclarr.yml` file and delete the generated files.
   
   How to combine them is documented here: https://recyclarr.dev/reference/config-examples/#merge-single-instance

   But I'll show how to combine them all in the next step for you. You can simply copy-paste it into your `recyclarr.yml` file.
