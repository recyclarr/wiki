This tutorial walks you through setting up Recyclarr to sync TRaSH Guide quality profiles to Radarr.
By the end, you'll have a working quality profile with custom formats and proper media naming.

:::info No YAML expertise required

You don't need to understand YAML structure or dig into template files. Recyclarr provides
ready-to-use configuration templates that mirror the TRaSH Guides exactly.

:::

## Prerequisites {#prerequisites}

- Radarr is running (Sonarr setup is similar; just use `sonarr` templates)
- API key ready (`Settings` → `General` → `Security`)
- Know your PUID/PGID:

  ```bash
  id -u  # PUID
  id -g  # PGID
  ```

## Step 1: Install Recyclarr {#install}

Add the Recyclarr service to your existing `docker-compose.yml` file, or create a new one.

```yml
services:
  recyclarr:
    image: ghcr.io/recyclarr/recyclarr:latest
    container_name: recyclarr
    user: 1000:1000 # Change to your PUID:PGID
    restart: unless-stopped
    # Set the time zone to match your location
    environment:
      - TZ=America/New_York
    volumes:
      - ./config:/config
```

For other installation methods, see the [Installation Guide](./installation/index.mdx).

## Step 2: Choose a Quality Profile {#choose-profile}

Recyclarr provides ready-to-use templates that match TRaSH Guide quality profiles. List them with:

```bash
docker compose run --rm recyclarr config list templates
```

You'll see available templates:

```txt
┌────────────────────┬─────────────────┐
│ Sonarr             │ Radarr          │
├────────────────────┼─────────────────┤
│ web-1080p-v4       │ hd-bluray-web   │
│ web-2160p-v4       │ uhd-bluray-web  │
│ anime-sonarr-v4    │ remux-web-1080p │
│ ...                │ remux-web-2160p │
│                    │ anime-radarr    │
│                    │ ...             │
└────────────────────┴─────────────────┘
```

These correspond to profiles on the [TRaSH Guides Radarr Quality Profiles][trash-radarr] page.

[trash-radarr]: https://trash-guides.info/Radarr/radarr-setup-quality-profiles/

## Step 3: Create Your Configuration {#create-config}

Generate a config file from your chosen template. For example, to create the `HD Bluray + WEB`
profile, run:

```bash
docker compose run --rm recyclarr config create --template hd-bluray-web
```

This creates `config/configs/hd-bluray-web.yml` with everything pre-configured.

To use this configuration:

- Open the file and update `base_url` and `api_key`:

  ```yml
  radarr:
    hd-bluray-web:
      base_url: http://radarr:7878  # Your Radarr URL
      api_key: your_api_key_here    # Radarr → Settings → General → API Key
  ```

- (Optional) Add these lines if you want Recyclarr to manage custom formats exclusively
  (recommended):

  ```yml
      delete_old_custom_formats: true
      replace_existing_custom_formats: true
  ```

That's it! The template already includes the quality definition, quality profile, and custom formats
from the TRaSH Guide.

Based on this output, add the `media_naming` block to your config file (under `api_key`). This
example uses `plex-tmdb` for both folder and movie naming:

```yml
    # Seen in Radarr > Settings > Media Management > Movie Naming
    media_naming:
      folder: plex-tmdb
      movie:
        rename: true
        standard: plex-tmdb
```

## Step 5: Run Your First Sync {#first-sync}

Now start the container in Cron Mode (runs sync automatically every day) and trigger the first sync
manually.

```bash
# 1. Start the container
docker compose up -d recyclarr

# 2. Trigger the sync
docker compose exec recyclarr recyclarr sync
```

You should see output like:

```txt
===========================================
Processing Radarr Server: [hd-bluray-web]
===========================================

[INF] Created 34 New Custom Formats
[INF] Total of 34 custom formats were synced
[INF] Created 1 Profiles: ["HD Bluray + WEB"]

[INF] Total of 14 sizes were synced for quality definition movie
[INF] Media naming has been updated
[INF] Completed at 12/25/2025 12:46:01
```

## Step 6: Verify in Radarr {#verify}

Check these locations in Radarr to confirm the sync worked:

- **Settings → Profiles**: Your new quality profile appears (`HD Bluray + WEB`)
  - (Optional) Delete other existing profiles if you don't need them
- **Settings → Custom Formats**: New custom formats with scores assigned
- **Settings → Quality**: Size ranges updated for each quality level
- **Settings → Media Management → Movie Naming**: Media naming formats applied

## When to Customize {#customize}

The templates provide the full TRaSH Guide recommendation. For most users, that's everything you
need.

You only need a `custom_formats:` section if you want to:

- Override a score for a specific custom format
- Add optional custom formats marked as "optional" in the TRaSH Guides.
  - For example, if you want releases with hybrid `DV + HDR10` to be preferred, simply uncomment
    that section in your config file.
  - See it in action in [Level Up setup](#power-user) below.

For customization examples, see the [Configuration Examples](../reference/config-examples.mdx) page.

## Adding More Profiles {#more-profiles}

Want multiple quality profiles (e.g., both 1080p and 4K) in a single Radarr instance? Add more
templates to your existing config file.

- Find the corresponding `include` templates for each of those profiles.

  ```bash
  docker compose run --rm recyclarr config list templates --includes
  ```

  Output (Radarr column only, language-specific and SQP IDs removed for brevity):

  ```txt
  ┌──────────────────────────────────────────────────────────────┐
  │ Radarr                                                       │
  ├──────────────────────────────────────────────────────────────┤
  │ radarr-custom-formats-anime                                  │
  │ radarr-custom-formats-hd-bluray-web                          │
  │ radarr-custom-formats-remux-web-1080p                        │
  │ radarr-custom-formats-remux-web-2160p                        │
  │ radarr-custom-formats-uhd-bluray-web                         │
  │ (...sqp custom-formats)                                      │
  │ radarr-quality-definition-movie                              │
  │ radarr-quality-definition-anime                              │
  │ (...sqp quality-definition streaming)                        │
  │ (...sqp quality-definition uhd)                              │
  │ radarr-quality-profile-anime                                 │
  │ radarr-quality-profile-hd-bluray-web                         │
  │ radarr-quality-profile-remux-web-1080p                       │
  │ radarr-quality-profile-remux-web-2160p                       │
  │ radarr-quality-profile-uhd-bluray-web                        │
  │ (...sqp quality-profiles)                                    │
  └──────────────────────────────────────────────────────────────┘
  ```

  Here it is visualized in a tree structure to see it better.

  To use multiple profiles, include both the `custom-formats` and `quality-profiles` templates:

  ```txt
  radarr
  ├── includes
  │   ├── custom-formats
  │   │   ├── sqp/...
  │   │   ├── radarr-custom-formats-anime.yml
  │   │   ├── radarr-custom-formats-hd-bluray-web.yml     <-- "HD Bluray + WEB"
  │   │   ├── radarr-custom-formats-remux-web-1080p.yml   <-- "Remux + WEB 1080p"
  │   │   ├── radarr-custom-formats-remux-web-2160p.yml   <-- "Remux + WEB 2160p"
  │   │   └── radarr-custom-formats-uhd-bluray-web.yml    <-- "UHD Bluray + WEB"
  │   │
  │   ├── quality-definitions
  │   │   ├── sqp/...
  │   │   ├── radarr-quality-definition-anime.yml
  │   │   └── radarr-quality-definition-movie.yml         <-- include once
  │   │
  │   └── quality-profiles
  │       ├── sqp/...
  │       ├── radarr-quality-profile-anime.yml
  │       ├── radarr-quality-profile-hd-bluray-web.yml    <-- "HD Bluray + WEB"
  │       ├── radarr-quality-profile-remux-web-1080p.yml  <-- "Remux + WEB 1080p"
  │       ├── radarr-quality-profile-remux-web-2160p.yml  <-- "Remux + WEB 2160p"
  │       └── radarr-quality-profile-uhd-bluray-web.yml   <-- "UHD Bluray + WEB"
  ```

- Move and rename the generated file `config/configs/hd-bluray-web.yml` from earlier to
  `config/recyclarr.yml`.
- Based on the tree above, populate the `include` section in your `recyclarr.yml` file:

  ```yml
  radarr:
    main:
      base_url: http://radarr:7878
      api_key: your_api_key_here
      delete_old_custom_formats: true
      replace_existing_custom_formats: true

      include:
        # Quality sizes (only needed once)
        - template: radarr-quality-definition-movie

        # Quality Profiles
        - template: radarr-quality-profile-hd-bluray-web
        - template: radarr-quality-profile-uhd-bluray-web
        - template: radarr-quality-profile-remux-web-1080p
        - template: radarr-quality-profile-remux-web-2160p

        # Custom Format Definitions
        - template: radarr-custom-formats-hd-bluray-web
        - template: radarr-custom-formats-uhd-bluray-web
        - template: radarr-custom-formats-remux-web-1080p
        - template: radarr-custom-formats-remux-web-2160p
  ```

:::tip The pattern

For each quality profile you want:

1. Add the `quality-profile` template (creates the profile)
2. Add the matching `custom-formats` template (adds CF scores to that profile)

The `quality-definition` template only needs to be included once.

:::

Run `docker compose exec recyclarr recyclarr sync` again, and all profiles will be created.

## Level Up: The "Power User" Config {#power-user}

Ready for the full setup? The configuration below is a "One and Done" setup. It includes:

- Radarr: 1080p, 4K, and Remux profiles
- Sonarr: 1080p and 4K profiles
- DV/HDR10+ support enabled

### Steps {#power-user-steps}

1. Make sure Radarr and Sonarr are running.
2. Check out the templates that you want to combine.

   ```bash
   docker compose exec recyclarr recyclarr config list templates
   ```

   These are the common ones for Radarr:
    - `hd-bluray-web`
    - `uhd-bluray-web`
    - `remux-web-1080p`
    - `remux-web-2160p`
    - `anime-radarr`

   And these for Sonarr:
    - `web-1080p-v4`
    - `web-2160p-v4`
    - `anime-sonarr-v4`
3. Create a config file for all the templates you want to combine.

   ```bash
   docker compose run --rm recyclarr config create --template hd-bluray-web
   docker compose run --rm recyclarr config create --template uhd-bluray-web
   docker compose run --rm recyclarr config create --template remux-web-1080p
   docker compose run --rm recyclarr config create --template remux-web-2160p
   docker compose run --rm recyclarr config create --template web-1080p-v4
   docker compose run --rm recyclarr config create --template web-2160p-v4
   ```

   This creates 6 files in the `config/configs` folder. Combine their contents into a single
   `config/recyclarr.yml` file and delete the generated files.

   See [Merging Configurations][merge-config] for the general approach, or use the ready-to-paste
   configuration in the next step.
